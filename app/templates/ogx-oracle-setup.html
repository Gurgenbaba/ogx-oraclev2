Voraussetzungen

Python 3.11+

OGX Oracle läuft lokal als Python-Server. Du brauchst Python 3.11 oder neuer.

# Version prüfen
python --version

# Download falls nicht installiert:
# https://www.python.org/downloads/

Außerdem brauchst du Tampermonkey im Browser:

Tampermonkey installieren
2
App starten

Server hochfahren

Entpacke den OGX Oracle Ordner und führe folgendes in PowerShell aus:

# Einmalig: Abhängigkeiten installieren
python -m venv .venv
.venv\Scripts\python.exe -m pip install fastapi uvicorn sqlalchemy aiosqlite pydantic pydantic-settings python-multipart jinja2

# App starten (danach immer so)
.venv\Scripts\python.exe -m uvicorn app.main:app --reload

Die App läuft dann unter http://127.0.0.1:8000

3
Collector Script installieren

Tampermonkey Script einrichten

1

Tampermonkey öffnen

Klicke auf das Tampermonkey-Symbol im Browser → Dashboard → Neues Script

2

Script einfügen

Den gesamten Inhalt der Datei ogx-oracle-collector.user.js in den Editor kopieren und speichern (Strg+S)

3

Im Dashboard sicherstellen dass das Script auf Aktiviert steht (grüner Toggle)

OGX aufrufen

Gehe zu uni1.playogx.com — im Menü erscheint automatisch der "OGX Oracle" Button unter "OGX Core"

4
API Key einrichten

Ingest absichern (optional für Localhost)

Der API Key verhindert, dass fremde Seiten Daten in deine Datenbank schreiben. Für Localhost-Betrieb ist er optional, aber empfohlen.

DEFAULT KEY svxlPbZBtZ9z79F8v0UIyeegaumjTJzy

Den Key im Spiel setzen:

→

Rechtsklick auf den "OGX Oracle" Button im Spiel-Menü → Key eingeben → Enter

→

Denselben Key in der .env Datei eintragen:
OGX_INGEST_API_KEY=dein-key-hier

Key niemals mit anderen teilen wenn du die App öffentlich betreibst.
Nutzung

Galaxy scannen

Öffne im Spiel die Galaxy-Ansicht. Das Script erfasst automatisch alle sichtbaren Planeten und sendet sie an OGX Oracle. Der Status wird neben dem "LOS!" Button angezeigt:

◉ Oracle   — bereit
✓ +5 ~2    — 5 neu importiert, 2 aktualisiert
⚠ 0 Rows  — leeres System (niemand da)
✗ Offline? — OGX Oracle Server nicht gestartet

Troubleshooting

Häufige Probleme

!

Badge zeigt "Offline?"

Der Python-Server läuft nicht. PowerShell öffnen und uvicorn starten.

!

Badge zeigt "API KEY?"

Rechtsklick auf den OGX Oracle Button im Menü → Key eingeben. Muss mit OGX_INGEST_API_KEY in der .env übereinstimmen.

!

OGX Oracle Button fehlt im Menü

Seite neu laden (F5). Das Script versucht 3x den Button einzufügen (0s, 0.8s, 2.5s nach Seitenload).

!

Badge zeigt "0 Rows"

Das System ist leer oder die Galaxy-Ansicht hat noch nicht vollständig geladen. Kurz warten oder manuell auf den Badge klicken.
